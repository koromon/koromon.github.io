遇到的问题：

根据教程发现有些测试无法通过，后来通过官方文档发现是由于测试工具升级导致的，随后根据官方教程步骤解决。

收获：

.babelrc

配置文件，用于把高级语法处理为目前大部分浏览器可以使用的语法。对于 presets 和 plugins，presets 可以看成是一堆的 plugins 插件预设，而 plugins 则是单独的插件，可以单独引用某个插件的功能，同时有些 presets 中没有提供的方法也可以在这里引入。

webpack 配置

  • entry 配置应用的入口
  • output 配置应用打包后的文件名和路径
  • module 配置相关的 loader，用于处理非 js 文件，把这些文件都处理为可以被 js 处理的有效模块。
  • plugins 也就是插件，用于执行范围更广的任务，涉及到整个构建流程。包括从打包优化和压缩，一直到重新定义环境中的变量。

前端测试

第一次比较直观的了解到断言测试，持续集成，性能测试以及 UI 测试这些测试方面相关的知识。

断言测试，也就是说我们通过设置期望得到的数值，然后通过代码进行测试得到反馈。通过某些插件，我们可以通过更加语义化的输入得到更好的反馈。而单元测试也就是针对某个模块和功能进行测试，也调用了断言库作为基础。

持续集成。如果没有持续集成，我们每次都要手动运行单元测试并构建项目，然后再人工监控反馈。但是进行持续集成后，我们每次提交代码，都会在远程仓库中自动执行上面的测试和构建的步骤，并且每次测试结果都有记录，大大的提高了效率。

性能测试，顾名思义。可以测试在不同的方法在各种环境下的运行效率，从而可以针对性的进行优化。

UI 测试，可以构建并渲染完后的页面，是否有自己预期的内容。并且还可以模拟里面的功能进行测试，看看结果是否和预期相符。
